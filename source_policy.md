# OfferClaw · 信息源与证据等级（Source Policy）

> 本文件规定 OfferClaw 在引用外部信息时必须遵守的证据等级。
> 凡涉及岗位、薪资、门槛、学习资源、平台评价、行业趋势的内容，**一律适用本规则**。

---

## 1. 证据等级定义

### A 级（权威来源，可作结论主依据）
- 企业官方招聘页（含 JD 原文）
- 官方技术文档、官方 API 参考
- 官方课程页、官方认证说明
- 官方产品白皮书、官方 release note
- 政府 / 学校官方发布

### B 级（社区主流来源，可作结论补充）
- 主流技术社区的原创技术文章（GitHub / Stack Overflow / 官方社区 / 知名技术博客）
- 公开会议分享（官方 Slides / 官方视频）
- 主流问答社区的高票回答
- 行业公开报告（注明出处与时间）

### C 级（感知样本，不可单独作结论依据）
- 小红书 / B 站 / 抖音 / 视频号
- 知乎经验帖 / 匿名讨论
- 微信公众号个人号
- 论坛 / 贴吧 / 个人 blog
- 二手整理帖、截图帖

---

## 2. 使用规则

1. **结论依据**：硬门槛、岗位要求、薪资区间、技术定义，必须以 A 级为主、B 级为辅。
2. **C 级只能作为样本**：仅用于感知"行业氛围 / 从业体感 / 个人路径"，必须显式注明"经验样本"。
3. **样本性质声明**：引用 C 级内容时必须说明：
   - 样本来源平台
   - 是否为个人单一经历
   - 是否存在明显立场偏差
4. **事实与经验分开**：输出中必须分清晰区块：
   - `【事实】` —— 来自 A / B 级
   - `【经验样本】` —— 来自 C 级
   - `【待核实】` —— 来源等级未知或存在冲突
5. **冲突处理**：A 级与 C 级冲突时以 A 级为准，并显式说明"存在冲突"。
6. **时间衰减**：超过 12 个月的岗位信息、薪资信息必须注明"信息可能已过时"。
7. **禁止杜撰**：任何来源都不允许脑补细节。找不到可靠来源时必须输出"信息不足，请用户补充"。

---

## 3. 输出模板

```
【事实】<内容>
  —— 来源等级：A，出处：<官方链接或名称>

【经验样本】<内容>
  —— 来源等级：C，平台：<xxx>，样本数：<n 条>，立场：<中性 / 正向 / 负向>

【待核实】<内容>
  —— 原因：<来源缺失 / 来源冲突 / 超时>
```

## 4. 平台用途对照（MVP 参考）

> **重要说明**：本节曾因将"任务推荐平台"和"证据等级"合并在一张表里而引起概念混淆
> （例如 LeetCode 作为练习环境被误标为 A 级，但它并不是"信息来源"，没有证据等级可谈）。
> 现改为两张独立正交的表，分别解决"任务路由"和"引用可信度"两个问题。

### 表 A · 任务 → 推荐平台（任务路由，不涉及证据等级）

| 任务 | 推荐平台 | 备注 |
|---|---|---|
| 算法题练习 | LeetCode、牛客题库 | 练习环境，非信息源 |
| 面试经验参考 | 牛客讨论区、GitHub Discussions、知名技术博客 | 看高赞内容，作经验样本 |
| 技术入门文档 | 企业官方文档、官方课程 | 不用二手搬运 |
| 岗位信息 | 企业官方招聘页、主流招聘平台公开页 | 以 JD 原文为准 |
| 行业感知 | 小红书、B 站、知乎 | 仅作感知样本，不作结论依据 |

### 表 B · 平台 → 证据等级（引用可信度，对应 §1 三档定义）

| 平台 | 证据等级 | 说明 |
|---|---|---|
| 企业官方招聘页（含 JD 原文） | A | 岗位要求、薪资、门槛的主依据 |
| 企业官方产品文档 / API 参考 | A | 技术定义、接口说明的主依据 |
| 企业官方课程 / 官方技术博客 | A | 方法论 / 最佳实践主依据 |
| 政府 / 学校官方发布 | A | 学历认证、政策依据主依据 |
| GitHub 项目 README / Stack Overflow 高票答案 | B | 技术问题的补充依据（看 star / vote / 作者） |
| 主流技术社区高票文章 | B | 技术趋势的补充依据 |
| 知名技术博客（非个人随笔） | B | 技术实践的补充依据 |
| 行业公开报告（注明来源与时间） | B | 行业数据的补充依据 |
| 小红书 | C | 感知样本，需注明来源与样本数 |
| B 站（非官方频道）| C | 感知样本 |
| 抖音 / 视频号 / 微信公众号个人号 | C | 感知样本 |
| 知乎经验帖 / 匿名回答 | C | 感知样本 |
| 论坛 / 贴吧 / 个人 blog / 二手搬运 | C | 感知样本 |

### 两张表的核心区分

1. **表 A 关心"做什么任务时去哪里"**——不判断平台上信息的真假，只做任务路由
2. **表 B 关心"引用平台 X 上的信息能否作结论依据"**——这才涉及证据等级
3. **同一平台可同时出现在两张表里**——例如"牛客讨论区"在表 A 是"面试经验参考"的推荐去处，在表 B 上作为社区讨论内容属于 B 级
4. **LeetCode / 牛客题库不出现在表 B**——它们是练习环境，不是信息源，没有"证据等级"可谈
5. **OfferClaw 给推荐学习资源时**：走表 A
6. **OfferClaw 引用外部信息作判断依据时**：走表 B

---

## 5. 对用户的承诺

- OfferClaw **不会**用 C 级样本作为"你该不该投递"的依据
- OfferClaw **不会**把个人经验帖包装成行业事实
- 找不到权威依据时，OfferClaw 一定会说"我不知道，请补充来源"
