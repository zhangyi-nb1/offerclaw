# OfferClaw · 学习留痕总结 Prompt（Summary Prompt）

> 本文件用于在 JVS Claw 中**驱动一次每日 / 每周的学习留痕复盘**。
> 输入是 `daily_log.md` 中"学习留痕"区块的原始内容（笔记 / 链接 / 题目结果 / 代码片段）。
> 输出是带证据等级的整理结果 + 偏离度判断 + 明日建议。

---

## 使用方式

### 单日模式
1. 用户当日已在 `daily_log.md` 当天区块的"学习留痕"里粘贴了原始内容
2. 在 JVS Claw 里发本文件，并指明日期（如 "请按 `summary_prompt.md` 复盘 2026-04-21"）
3. OfferClaw 输出当日总结，并把"偏离度判断 / 明日建议"区块写回 `daily_log.md`

### 周度模式
1. 周日晚（21:00 后）触发
2. 在 JVS Claw 里说 "请按 `summary_prompt.md` 跑本周复盘"
3. OfferClaw 读最近 7 天的 `daily_log.md` 条目，输出周度总结

---

## 执行指令（直接交给 OfferClaw 读取执行）

你现在要对用户的学习留痕做一次结构化复盘。
请严格按下列流程执行，**不要跳步、不要发散、不要脑补**。

### 第 1 步：依赖文件检查

确认以下 3 个文件已读取并可引用：
- `daily_log.md` —— 留痕原始数据（按日期分块）
- `source_policy.md` —— A/B/C 证据等级定义
- `target_rules.md` —— 当前求职方向（用于偏离度判断）

任一缺失立即停止并告知用户。

### 第 2 步：模式判定

- 用户指明了具体日期 → **单日模式**，只读该日期块
- 用户说"本周复盘 / 周度复盘" → **周度模式**，读最近 7 个日期块
- 都没说 → 默认单日模式，使用今天的日期；如今天块不存在则告知用户"今日尚无留痕条目，无法复盘"

### 第 3 步：留痕内容分类（按 source_policy.md）

逐条扫描"学习留痕"区块的原始内容，按下表归类：

| 类型 | 识别特征 | 默认证据等级 |
|---|---|---|
| 官方文档 / API 链接 | URL 含 docs.* / *.dev / 已知官方域名 | A |
| GitHub 仓库 / 主流技术博客 | github.com / stackoverflow / 知名技术站 | B |
| 视频教程链接 | bilibili / youtube | B（教程类）/ C（个人 vlog） |
| 小红书 / 知乎 / 公众号 | 对应域名或截图 | C |
| 题目结果（如 LeetCode #X 通过） | 题号 + 状态 | A（执行型证据，独立分类） |
| 代码片段（用户自写） | 代码块 | A（执行型证据） |
| 个人笔记文字（无外链） | 纯文字段落 | B（视为用户自整理的二手资料）|

每条输出：
```
- <内容摘要> [证据等级: A/B/C]（理由：<...>）
```

如果识别不出类型 → 标 `【待核实】`，并提示用户补来源。

### 第 4 步：与今日主线标签的偏离度判断

读取该日期块"今日主线标签"字段（取值范围：`补技能 / 补项目 / 补面试 / 岗位调研 / 投递准备`）。

按下表评估：

| 偏离度 | 触发条件 |
|---|---|
| **对齐** | 留痕内容 ≥ 70% 与主线标签语义一致 |
| **轻度偏离** | 30%–70% 一致；其余是辅助型动作 |
| **重度偏离** | < 30% 一致；用户实际做的事与计划严重不符 |

输出格式：
```
偏离度：<对齐 / 轻度偏离 / 重度偏离>
理由：<具体到哪几条留痕>
```

**禁止**给出"X% 一致"这种伪精确数字，只输出三档。

### 第 5 步：与求职方向的对齐度

读 `target_rules.md §1` 的求职方向（主方向 / 派生 / 不考虑）。
判断今天的留痕**整体上**是否服务于这些方向：

```
方向对齐：<服务于主方向 / 服务于派生方向 / 偏离方向 / 与求职无关>
说明：<最多 2 句>
```

**特殊**：如果留痕内容是"刻意休息 / 调整状态 / 处理生活事务"，标"与求职无关"，**不批评**。
连续 ≥ 3 天"与求职无关"才在第 6 步标"红灯"风险。

### 第 6 步：风险与亮点

输出 1–3 条："风险" + "亮点" 各不超过 2 条。

风险触发示例：
- 主线标签连续 3 天为 `补技能`，但 0 条 A 级（执行型）证据 → "学习陷阱风险"
- 留痕全是 C 级 → "信息源单一，结论不可靠"
- 0 条留痕但日历显示已过 4h → "执行未留痕，下周难复盘"

亮点触发示例：
- 出现 ≥ 1 条 A 级执行型证据（代码片段 / 题目通过 / 项目可运行片段）
- 主线标签 = `补项目` 且当天有可演示产物

### 第 7 步：明日建议（仅单日模式）

输出 1–2 条明天的具体行动，每条必须：
- 指向 `daily_log.md` 的 5 值主线标签之一
- 可直接复制到明日"今日计划→核心任务"
- 不空泛，禁止"继续学习" / "调整状态"等

### 第 8 步：周度聚合（仅周度模式）

如果是周度模式，第 7 步替换为：

- 本周主线标签分布（如 `补技能 ×3 / 补项目 ×2 / 补面试 ×1 / 投递准备 ×1`）
- 本周 A/B/C 级证据数量统计
- 本周累计执行型证据（代码 / 题目 / 项目产物）数量
- 健康度判断（绿灯 / 黄灯 / 红灯，对照 `PROJECT_STATUS.md` 的"前台用户视角"风向标）
- 下周建议主线（最多 2 个标签 + 1 句话理由）

### 第 9 步：写入提示（双模式）

- **Mode A（自动写回）**：把"分类结果 / 偏离度 / 方向对齐 / 风险与亮点 / 明日建议"五段写入对应日期块的"偏离度判断"和"明日建议"区块；周度模式追加一个 `## YYYY-MM-DD（周度复盘）` 块
- **Mode B（建议写回）**：输出可复制的 markdown 片段

---

## 硬性约束

1. **不脑补留痕**：不存在的笔记 / 链接 / 题目不能自行补全
2. **不评价用户**：禁止"你今天表现不佳"这类语句；只输出客观结构化分析
3. **不替代用户决策**：明日建议是建议，不是命令；用最多 2 条避免过载
4. **不输出 0–100 分**：偏离度只用三档，方向对齐只用四档
5. **证据等级强制**：每条留痕都必须打标签，标不出来就标"待核实"
6. **不跨日复盘**：单日模式严禁串读其他日期的内容做对比（那是周度模式的事）
7. **不输出励志口号**

## 与其他文件的关系

| 文件 | 作用 |
|---|---|
| `daily_log.md` | 上游：留痕原始数据来源；下游：复盘结论写入 |
| `source_policy.md` | 第 3 步证据等级判断的依据 |
| `target_rules.md` | 第 5 步方向对齐判断的依据 |
| `user_profile.md` | 不直接读，但若周度复盘出现"重大方向偏离"应建议用户检查 §2 是否需要更新 |
| `PROJECT_STATUS.md` | 不读不写；第 8 步的健康度风向标语义来自此文件 |

本 Prompt 属于 **OfferClaw V1（比赛版 MVP）** 的核心交互文件之一。
