# OfferClaw · 真实投递追踪（Applications Tracker）

> 用途：记录**真实**的求职投递与反馈进展。
> **概念隔离**：本表是**投递池**（推动真实进展，状态机驱动）；
> [`jd_candidates.md`](jd_candidates.md) 是**测试池**（用于 `match_job`/`eval_rag` 回归对比，不代表已投）。
> 同一岗位可两边都有：先入 candidates 评估 → 决定投 → 同步入本表。
> 写入规则：见 `DATA_CONTRACT.md` §4.1。状态变更必须由用户确认，**OfferClaw 不做自动投递**（见 `docs/ethical_use.md`）。
> 联系方式一律**脱敏**（HR 写姓+尾号 4 位即可）。

## 状态枚举（**只能从下表选**）

| 状态 | 说明 | 下一步默认动作 |
|---|---|---|
| `已评估` | OfferClaw 已分析过这个岗位，但还没决定投不投 | 决定投/不投 |
| `准备投递` | 决定投，正在改简历 / 写求职信 | 投递 |
| `不投递` | 主动放弃（与方向/地点/时间不匹配） | — |
| `已投递` | 简历已发出 | 等反馈 |
| `等待反馈` | 投递 ≥7 天无回复 | 跟进或放弃 |
| `面试中` | 收到面试邀请 | 准备 |
| `已 Offer` | 收到 Offer | 评估接受 |
| `已拒绝` | 公司拒绝 | 复盘 |
| `主动放弃` | 自己中途退出 | 复盘 |

## 投递 SOP（每条投递跑一遍）

1. **读 JD 全文** → 复制摘录到 `jd_candidates.md` 对应段
2. **OfferClaw 跑匹配** → `python match_job.py`（或 `/api/match`）→ 三档结论
3. **如为"适合"** → 在本表加一行，状态 `已评估`
4. **改简历** → 参考 `docs/resume_pitch.md`，按 JD 关键词突出最相关 2-3 段经历；状态 `准备投递`
5. **投递** → 状态 `已投递`；备注里写简历版本号（如 `resume_v3_nio.pdf`）
6. **跟进** → 7 天无回复改 `等待反馈`；收到面试改 `面试中`，并在 `interview_story_bank.md` 准备 STAR
7. **结果** → `已 Offer` / `已拒绝` / `主动放弃`；任何终态都要在 `summaries/` 写一份单岗位复盘
